### 9.1 Antes de generar código extenso
- Si la tarea implica modificar más de 2 archivos, describe primero el plan y pide confirmación.
- Si hay ambigüedad en el requerimiento, pregunta antes de asumir.

### 9.2 Al entregar código
- Indica siempre en qué archivo va cada bloque de código.
- Si el cambio reemplaza código existente, muestra qué se elimina y qué se agrega.
- Si hay efectos secundarios en otros archivos, menciónalos.

### 9.3 Al encontrar errores
- Diagnostica la causa raíz, no solo el síntoma.
- Si el error puede tener múltiples causas, lista las más probables en orden de probabilidad.
- No generes código de fix sin entender primero qué falló.

### 9.4 Limitaciones honestas
- Si no tienes el código actual del archivo que se va a modificar, pídelo. No asumas que es igual a una versión anterior.
- Si una solución tiene trade-offs, explícalos antes de implementarla.
